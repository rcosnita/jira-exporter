RAIN Rendering Engine

Rendering engine is the heart of RAIN. It's the mechanism that makes all components processing / rendering non blocking. For an example imagine the following scenario: you have a dashboard with thirty widgets added on it. Rendering engine will do the following processing:

  • load the aggregated markup of the dashboard

  • each component which requires server side data will be replaced by placeholders that behaves like proxy. From Customer perspective this mean that the customer will receive feedback from application as fast as possible (response time = rendering time which today ~ 50ms).

  • All the components arrive on the client when they are loaded. This will give customer a progressive loading behavior of the application which ensures continuous feedback.

RAIN SDK

RAIN SDK is a command line tool that facilitate developers easy access to all RAIN features. In the SDK for 1&1 there will be different processes implemented like Technical editors translate / release or Release Managers release process. Each operation from RAIN SDK will provide help man pages for easy usage.

There is a good idea to see how Android SDK or Java SDK is structured and what operations are available. In addition, for best results in deciding command line arguments and how it can be invoke see gcc or g++ linux command lines.

RAIN Remoting

RAIN is real modular and distributed platform. RAIN resources (components) can be spreaded across multiple unbalanced RAIN nodes and still can be aggregated within a single application. For this to work remote components are a core feature of RAIN. When aggregating components that do not run in the same container (remote components) the following requirements need to be taken into consideration:

  • RAIN must keep track of remote fragments session.

  • RAIN must send aggregated page parameters to each remote fragment.


More information about remote components are described on feature proposal: http://rainjs.github.com/rainjs/proposals/Features-proposal-remote-aggregation.html
